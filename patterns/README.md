# Patterns Library

Token-efficient reference for high-performing hooks, story structures, tone patterns, and payoff styles.

**Load only when explicitly needed. Never load the full patterns during every cycle.**

---

## Files

- `PATTERNS.md` — All pattern libraries (token-optimized, short format)

---

## When to Load

- During Concept step: generating or auditing hooks
- During Strategy step: auditing generic angles
- Checking tone consistency
- No other time

---

## When NOT to Load

- During ideation
- Before concept generation (generate first, reference if needed)
- Every cycle as routine
- For documentation purposes

**Loading on demand reduces token usage by ~30% per cycle.**

---

## Pattern Categories

1. **Self-Recognition Hooks** — Audience sees themselves in <3 seconds
2. **Reframe Hooks** — Shift perspective on something familiar
3. **Adaptive Hooks** — Context-specific, generated per angle
4. **Story Structure Patterns** — Problem, Pursue, Payoff frameworks
5. **Tone Patterns** — Voice, specificity, realism standards
6. **Payoff Styles by Mission** — Mission-specific endings

---

## Example: Using Self-Recognition Hooks

During Concept generation, if generating hooks:

1. Open `PATTERNS.md` → Self-Recognition Hooks section
2. Pick a pattern that matches this angle's emotional core
3. Adapt it to this specific audience + angle
4. Test against Recognition + Clarity

Result: Faster generation, higher quality, consistent voice.

---

## Maintenance

Update after each content cycle if:
- A new hook pattern emerges and performs well
- A story structure repeatedly succeeds
- Tone patterns shift based on feedback

Document the update with date and reason.
