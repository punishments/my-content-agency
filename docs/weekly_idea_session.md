# Weekly Idea Session

Monday SOP. Runs before any proposals begin. Takes the week from zero to 6 ideas assigned to slots and ready for the proposal pipeline.

This is not a creative session — it is a planning session. The creative work happens in the pipeline. This session decides what goes into the pipeline and in what order.

---

## When to Run

Every Monday before any content work begins. Target: 60–90 minutes total.

**Prerequisites — have these open before starting:**
- `planning/weekly_plan.md` — to check what slots are open and what carried over
- `planning/content_backlog.md` — to check what ideas are already queued
- `inputs/idea_dump.md` — to pull any ideas captured mid-week
- `docs/content_decision_system.md` — extraction questions are in there

---

## Step 1 — Load the Week (10 min)

Before generating anything, understand the state of this week.

**Check weekly_plan.md:**
- Are any slots already filled from last week's carryovers?
- What is the current distribution? (Mission / Format / Intensity)
- How many open slots remain? That is the number of new ideas needed.

**Check content_backlog.md:**
- Are there any High priority ideas already in the backlog ready to schedule this week?
- If yes — pull them into open slots before generating new ideas. They've already passed triage.

**Check idea_dump.md:**
- Any raw ideas captured since last Monday that haven't been assessed yet?
- Pull these into the generation pool.

**Output:** A count of open slots + a clear view of the current week's distribution constraints.

---

## Step 2 — Generate Ideas (20 min)

Using the extraction questions in `docs/content_decision_system.md`, generate enough raw ideas to fill open slots plus 2–3 extras.

**Target:** If 6 slots are open, generate 8–9 ideas minimum. Extras give you triage flexibility.

**Rules:**
- Write fast — one sentence per idea
- Don't filter during generation
- Pull from experience, struggle, learning, and backlog before reaching for research
- Do not try to fit an idea to a slot yet — generate first, route after

---

## Step 3 — Triage Each Idea (2 min per idea)

For each idea, run this filter before committing it to a full proposal. This is the gate between idea and pipeline investment.

**3-question triage:**

| Question | Pass | Fail → Action |
|----------|------|---------------|
| Is there a genuine brand angle? Does it connect to a belief, pillar, or real audience pain? | Yes — proceed | No — park in backlog or drop |
| Would my audience stop scrolling? Is the recognition specific enough? | Yes — proceed | Too vague — sharpen or drop |
| Do I have the material and energy to execute this this week? | Yes — proceed | No — backlog |

An idea needs to pass all 3 to move forward this week. If it fails one, route it to the backlog with a note on what's missing, or drop it entirely.

**Do not spend more than 2 minutes per idea.** If you're still uncertain after 2 minutes, it's not ready — park it.

---

## Step 4 — Route Ideas (10 min)

After triage, assign each surviving idea to a slot or the backlog.

**This week → if:**
- It passed all 3 triage questions
- A slot exists for its mission and format
- Intensity fits what the week allows

**Backlog → if:**
- It passed triage but the slot it needs is already filled
- It's not time-sensitive and a later week is more appropriate
- It needs more material before it can be executed

**Drop → if:**
- It failed triage and has no clear path to a genuine brand angle
- The energy isn't there — a forced idea produces forced content

**After routing:** Assign each this-week idea a preliminary slot in `planning/weekly_plan.md` (Mission / Format / Intensity). Do not start proposals yet — confirm the full week's balance first.

---

## Step 5 — Confirm Weekly Balance (10 min)

Before opening any proposals, verify the full week's slot assignments hold together.

Run the distribution check:

| Dimension | Target | Assigned | Remaining |
|-----------|--------|----------|-----------|
| Attract | 4 | | |
| Nurture | 1 | | |
| Position | 1 | | |
| Talking | 3 | | |
| Non-talking | 3 | | |
| High intensity | 2 | | |
| Low intensity | 4 | | |

**If the balance is off:** Swap or replace ideas now — before any pipeline work begins. It is much easier to reorder ideas at this stage than to return a completed proposal to Step 3.

**Common imbalances to catch here:**
- 5 Attract ideas selected, no Nurture or Position
- All 2 High intensity slots used on the same format
- 3 Talking posts with no Non-talking planned yet

---

## Step 6 — Open Proposals in Sequence (rest of Monday–Tuesday)

Once the week is balanced, begin proposals using `docs/content_pipeline.md` Steps 1–5.

**Recommended proposal order — run the constrained slots first:**

1. Position (1 slot — fills up fastest, hardest to reassign)
2. Nurture (1 slot — personal content, often needs the most creative thought)
3. Attract × 4 (more flexible — fill after the locked slots are solid)

This prevents arriving at post 5 or 6 and discovering the one remaining Nurture slot is the wrong fit for every remaining idea.

**Each proposal:** Complete all Steps 1–5 before moving to the next. Do not run them in parallel — the distribution check in Step 3 of each proposal depends on what the previous proposals already claimed.

---

## Output

By end of Monday–Tuesday:

- [ ] 6 ideas triaged and slot-assigned in `planning/weekly_plan.md`
- [ ] 6 complete proposals saved to `planning/proposals/`
- [ ] `planning/weekly_signoff.md` populated and ready for your review
- [ ] Backlog updated with any ideas that didn't make this week

---

## Rules

- **Never skip the balance check.** One unbalanced week is easy to fix before proposals begin. It is disruptive to fix after.
- **Backlog carryovers get first claim on slots.** Ideas already in the backlog have been vetted. New ideas compete against them, not the other way around.
- **Do not start proposals until the full 6-slot balance is confirmed.** Starting proposals with an unresolved balance wastes pipeline work.
- **Energy is a triage criterion.** An idea you are not interested in will produce content your audience is not interested in. If the energy isn't there, park it.
